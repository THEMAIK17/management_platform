Documento Técnico - Management Platform

1. Estructura del Proyecto

El sistema está desarrollado bajo el patrón Clean Architecture, dividiendo la lógica en 4 capas principales para asegurar mantenibilidad y escalabilidad:

  • Domain (Núcleo): Contiene las entidades (Project, TaskItem), enumeraciones y excepciones de negocio.
  • Application: Orquesta la lógica de negocio mediante servicios y define los DTOs.
  • Infrastructure: Implementa el acceso a datos con EF Core + PostgreSQL y la seguridad.
  • API / Web: Capas de presentación MVC y RESTful.

2. Descripción de Carpetas y Módulos

  • Platform.Domain/Entities: Modelos de datos fundamentales.
  • Platform.Application/DTOs: Objetos de transferencia de datos.
  • Platform.Infrastructure/Data: Configuración de base de datos y migraciones.
  • Platform.Api/Controllers: Manejo de endpoints.
  • Platform.Web/Views: Interfaz de usuario basada en Razor.

3. Flujo del Sistema

El flujo de información sigue un camino unidireccional:

Diagrama de Flujo: Entradas → Controlador → Servicio → Dominio → Persistencia (DB)

  1. Entrada: El usuario realiza una acción en la web.
  2. Controlador: Recibe la petición y valida la sesión.
  3. Servicio: Ejecuta la lógica delegada.
  4. Dominio: Valida reglas de negocio críticas.
  5. Persistencia: EF Core guarda los cambios en PostgreSQL.

4. Fragmentos de Código Relevantes

Regla de Negocio: Activación de Proyecto

public void Activate(int taskCount)
{
    if (taskCount == 0)
        throw new DomainException("El proyecto debe tener al menos una tarea.");

    Status = ProjectStatus.Active;
}
