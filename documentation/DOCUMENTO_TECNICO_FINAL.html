Proyecto: Management Platform (Gestión de Proyectos y Tareas)

1. Estructura del Proyecto

El sistema se ha diseñado siguiendo el patrón de Arquitectura Cebolla (Clean Architecture), lo que permite un desacoplamiento total de la lógica de negocio respecto a las herramientas externas (bases de datos, frameworks, etc.).

Management_Platform.sln
├── Platform.Domain // Capa 1: Entidades y reglas de negocio puras
├── Platform.Application // Capa 2: Casos de uso y DTOs
├── Platform.Infrastructure // Capa 3: Persistencia, Auth y Servicios Externos
└── Platform.Web / API // Capa 4: Presentación (MVC / REST)

2. Descripción de Carpetas y Módulos

  • Platform.Domain: Contiene el modelo de datos principal (`Project`, `TaskItem`, `User`) y las interfaces de los repositorios que definen el contrato de datos.
  • Platform.Application: Implementa los servicios que coordinan las operaciones. Utiliza el patrón Unit of Work para asegurar la integridad de las transacciones.
  • Platform.Infrastructure: Gestiona la conexión a PostgreSQL mediante Entity Framework Core. Aquí se encuentra la lógica de seguridad (JWT y encriptación BCrypt).
  • Platform.Web: Provee la interfaz de usuario final utilizando ASP.NET Core MVC y vistas Razor dinámicas.

3. Explicación del Flujo del Sistema

El flujo del sistema es modular y seguro:

USUARIO → CONTROLADOR (Auth) → SERVICIO (Lógica) → DOMINIO (Reglas) → REPOSITORIO (DB)
  1. El usuario interactúa con la interfaz web para realizar una acción (ej. Crear una tarea).
  2. El controlador valida la identidad del usuario (vía Cookies o JWT).
  3. Se invoca un servicio de aplicación que recibe los datos filtrados (DTOs).
  4. El servicio consulta las entidades de dominio para validar si el cambio es permitido (ej. no añadir tareas a un proyecto completado).
  5. Si todo es correcto, la infraestructura persiste los datos en la base de datos PostgreSQL.

A. Regla de Negocio: Validación de Activación

// Ubicado en Platform.Domain/Entities/Project.cs
public void Activate(int taskCount) {
    // REQUISITO: Validar que el proyecto no esté vacío para poder iniciarlo
    if (taskCount == 0)
        throw new DomainException("El proyecto debe tener al menos una tarea para activarse.");

    this.Status = ProjectStatus.Active;
}

B. Seguridad: Encriptación de Contraseñas

// Ubicado en Platform.Infrastructure/Services/AuthService.cs
public async Task RegisterAsync(string email, string password) {
    // Uso de algoritmo de hashing seguro (BCrypt) antes de persistir
    var passwordHash = BCrypt.Net.BCrypt.HashPassword(password);
    var user = new User(email, passwordHash);
    await _uow.Users.AddAsync(user);
}
